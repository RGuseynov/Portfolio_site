Etude du marché immobilier français

Ce projet est constitué de deux éléments :

  • Une cartographie du prix du m2 par département pour les appartements en France
  • Un estimateur de prix pour appartement

L'ensemble du code du site et du projet est disponible sur GitHub

Le projet a été développé en Python avec Flask, Pandas, Scikit-learn et Folium

Données source :

  • DVF(Demandes de valeurs foncières) géolocalisées contenant l'historique des transactions foncières depuis 2014
  • GeoJSON de la France contenant la délimitation des départements

Etapes de traitement des données:

  • Exploration des données (notamment de leurs distributions)
  • Nettoyage/Filtrage (notamment des valeurs abérantes assez nombreuses), je garde les features (colonnes/dimensions) pértinentes pour la suite
  • Aggrégation des données (pour connaitre le prix du m2 pour chaque département pour chaque année)
  • Association de ces données au GeoJSON et génération de la carte de prix
  • Préparation et sauvegarde des données propres pour faire du machine learning

Machine Learning :

En utilisant la surface, le nombre de pièces et la géolocalisation du bien: entrainement d'un modèle de régression linéaire et d'un arbre de décision.

  • Pour la Régression linéaire : de part la corrélation non linéaire entre les features de géolocalisation et le prix du bien, il n'est pas possible de les utiliser tel quel. Utilisation du prix médian des départements comme feature de géolocalisation. Standardisation des données.
  • Pour l'arbre de décision : utlisation de différentes combinaisons de features de géolocalisation parmis le département, le code postal, le code commune et le duo longitude/latitude. Il n'y a pas de grande différence, hormis le fait que le code département a peu d'impact sur le modèle étant donné qu'on dispose de "versions" plus précise de cette feature que sont le code postal et le code commune. L'utilisation du duo longitude/latitude n'augmente pas la précision du modèle.

Dans tout les cas, la surface, le nombre de pièces et la géolocalisation ne sont pas suffisants à eux seuls pour estimer correctement un bien. Beaucoup d'autres variables dont on ne dispose pas peuvent avoir des impacts significatifs sur le prix (ancienneté, état du bien, chauffage, étage...). On pourrait aussi penser à croiser la localisation des biens avec d'autres données geospatiales (densité des zones, axe routier, proximité transport/commerce, distance de la mer, altitude...).

Résultats :

Mauvais pour le modèle de régression linéaire et relativement correcte pour l'arbre de décision en considérant le faible nombre de features.

Le pourcentage d'écart moyen entre le prix réel et le prix estimé est de 62% pour la régression linéaire et de 28% pour l'arbre de décision.
